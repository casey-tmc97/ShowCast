ShowCast

Real-time broadcast graphics and presentation software. Build slides, drive live outputs, and automate your production workflow.

Overview

ShowCast combines slide design, live output control, rundown management, and scheduling into a single application. It is designed for broadcast, live events, and houses of worship that need reliable, flexible graphics delivery.

Key Capabilities

  • Multiple simultaneous outputs — drive different content to different screens or NDI streams at once
  • Layer-based slide editor — background, text, image, and shape layers with per-layer animations
  • Rundown view — organize packages by role (lower-third, full-screen, etc.) and output assignment
  • Timers — countdown and wall-clock timers that can be embedded in slides or triggered automatically
  • Scheduler — fire packages at exact times, with daily/weekly/monthly recurrence and timer actions

Shows & Packages

Shows

A Show is the top-level container for your production. It holds one or more packages. ShowCast works with a single show file (.showcast) that is automatically saved and reloaded on startup.

Packages

A Package is a named collection of pages. Think of it as a slide deck — a song, a lower-third set, a countdown package, a bumper loop.

  • You can have as many packages as you need
  • Packages can be assigned to specific outputs in the rundown
  • The active package for an output is shown in the center page grid

Creating & Managing Packages

Use the package browser in the left panel to create, rename, and switch between packages. Click a package name to load it into the active output's page grid.

Pages & Layers

Pages

A Page (also called a slide) is a single graphic frame within a package. Pages are ordered and displayed as thumbnail cards in the center panel.

Page options (right-click any page card)

OptionDescription
EditOpen the slide editor for this page
Go LiveSend this page to the active output immediately
Set Go-to-Next Timer…Auto-advance to the next page after N seconds
Remove TimerClear the auto-advance timer
Trigger → Timer → [name]Play a timer when this page goes live (toggle per timer)
Default TransitionOverride the output's default transition for this page
DuplicateCopy the page and insert it immediately after
DeleteRemove the page from the package

Layers

Each page is composed of layers stacked by Z-order. Open the editor to add, remove, and style layers.

Layer Types

TypeDescription
BackgroundSolid color fill that covers the entire slide
TextStyled text block. Can be linked to a live timer via Timer Binding
ImageBitmap image from disk. Supports Fit, Fill, Stretch, and Center modes
ShapeRectangle or rounded rectangle with optional stroke

Layer Animations

Each layer supports independent Entry and Exit animations with configurable duration, delay, and easing. Animations play when the page transitions in or out.

Layer Roles

Layers can be assigned to specific output roles so that the same package can render differently on different outputs (e.g., a confidence monitor versus a projector).

Timer Binding

A text layer can be bound to a Timer. When bound, the layer's text is replaced in real time with the timer's current display value. This lets you embed a countdown directly on a slide.

  1. Select a text layer in the editor
  2. In the Inspector, choose a timer from the Timer Binding dropdown
  3. The layer will show live timer text on all outputs

Outputs

An Output represents a display destination — a monitor, projector, or NDI stream. Each output independently tracks which package and page is live.

Setting Up Outputs

Go to ShowCast → Settings → Screens to configure your outputs. Each output has:

  • Name — display label
  • Resolution — width × height in pixels
  • Screen assignment — which physical monitor to use
  • NDI — optionally stream via NDI instead of (or in addition to) a physical display

Selecting the Active Output

Use the output selector in the left panel. The center page grid and preview always reflect the currently selected output.

Output Preview

The live preview in the bottom-left corner shows exactly what is on screen for the selected output, including the current transition state.

Going Live

Basic Methods

ActionResult
Click a page thumbnailGoes live immediately on the selected output
SpaceAdvance to the next page (or fire a pre-cued page)
↑ / ↓ or ← / →Move selection without going live (pre-cue)
ReturnGo live with the currently selected page
Right-click → Go LiveGo live with a specific page

Pre-Cueing

Use arrow keys to select a page without firing it. The selection highlight shows what will go live next when you press Space or Return. This lets operators cue the next slide while the current one is still live.

Clear Live

Press the Clear button or use the keyboard shortcut to take the output to black, removing all live content without transitioning to a new page.

Transitions

Transitions control how one page replaces another on an output.

TypeDescription
CutInstant switch — no animation
FadeCross-dissolve between outgoing and incoming page
WipeDirectional reveal
ZoomScale-in effect
Slide Left / RightHorizontal slide transition

Setting Transitions

  • Output default — set in the output configuration; applies to all pages unless overridden
  • Per-page override — right-click a page → Default Transition to set a different transition for that specific page
  • Next transition — use the transition selector in the toolbar to change the transition for the next fire only

Rundown

The Rundown organizes your packages in show order and assigns each package to a specific output. It is the primary workflow for a structured live production.

Rundown View

Switch to Rundown mode using the view toggle at the top of the center panel. The page grid changes to show packages grouped by their position in the rundown, with the output assignment visible for each group.

Rundown Entries

Each entry in a rundown specifies:

  • Package — which package to display
  • Output — which output to send it to
  • Label — optional custom name for the entry

Firing from the Rundown

Click any page thumbnail in the rundown view to send it live on that group's assigned output. The spacebar and arrow key navigation works the same as in the flat page grid.

Timers

Timers appear in the Timer Panel (left sidebar). They run independently of slides and can be displayed on-screen by binding them to text layers.

Timer Types

TypeDescription
Counter Counts down (or up) between two values. Supports a warning color when approaching the end, and overflow to continue counting past zero.
Clock Counts down to a specific wall-clock time (e.g., "until 11:00"). Always running — shows the time remaining until the next occurrence of that time each day.

Timer Controls

ButtonAction
▶ / ⏸Play / Pause the timer
⏹Reset to start value
✎Edit timer settings
✕Delete the timer

Counter Settings

  • From / To — start and end values in M:SS format
  • Warn — show an orange color when N seconds remain
  • Overflow — continue counting past the end value (turns red)

Display Colors

ColorMeaning
WhiteNormal running state
OrangeWarning — within the warn threshold of the end value
RedOverflow — past the end value

Scheduler

The Scheduler fires packages automatically at specified times, with support for recurring schedules and timer actions. Access it via ShowCast → Settings → Schedule.

Calendar View

The left panel shows a monthly calendar. Days with scheduled events display an orange dot. Click a day to see its events listed on the right.

Scheduled Events

Each event specifies:

  • Date & Time — when to fire
  • Rundown — which rundown to use for output assignment
  • Package — which package to go live
  • Page — optionally, a specific page within the package (defaults to page 1)
  • Label — optional display name

Recurrence

TypeOptions
NoneOne-time event — fires once and is marked done
DailyEvery N days
WeeklyEvery N weeks on selected days of the week
MonthlyEvery N months on the same day of the month

All recurring events support an optional Until end date.

Timer Actions

Each scheduled event can also control timers when it fires. Use the Timer Actions section in the Add/Edit Event dialog to add one or more timer actions:

  • Play — start the timer (does nothing if already running)
  • Pause — pause the timer
  • Reset — reset to start value without playing
  • PlayPause — toggle between play and pause
Tip: Combine a scheduled package fire with a timer Reset + Play to automatically start a countdown whenever a specific program element begins.

Page Triggers

A Page Trigger lets a specific page automatically control a timer the moment it goes live — no manual interaction required.

Setting a Trigger

  1. Right-click a page card in the page grid
  2. Hover over Trigger → Timer
  3. Click a timer name to toggle it on (✓) or off

A page can trigger multiple timers simultaneously.

Trigger Behavior

Timer State When Page Goes LiveAction
Already runningNo action — timer continues uninterrupted
Paused mid-countResume from current position
Stopped / at startPlay from current position
Note: Triggers fire on every go-live method — clicking a thumbnail, pressing Space, pressing Return, right-click → Go Live, and automated scheduler fires.

Common Use Cases

  • A "sermon begins" slide that starts a speaker timer
  • A countdown package page that starts a clock timer for a live segment
  • An intermission slide that resumes a pre-paused timer
